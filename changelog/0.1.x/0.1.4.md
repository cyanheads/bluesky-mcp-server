# 0.1.4 — 2026-06-15

Adopts `@cyanheads/mcp-ts-core` `^0.9.21 → ^0.10.6`: server identity in `initialize`/`.well-known`, the `ctx.enrich.truncated()` disclosure helper (0.10.2), and the entrypoint-identity + post-bundle packaging checks (0.10.6).

## Added

- **Server identity** — `createApp()` now sets `name: 'bluesky-mcp-server'` and `title: 'bluesky-mcp-server'`, populating `initialize` serverInfo and `/.well-known/mcp.json` with the unscoped repo name.
- **Page-truncation disclosure** — `bsky_search_posts`, `bsky_search_actors`, `bsky_get_author_feed`, `bsky_get_follows`, and `bsky_get_trending` declare `truncated` / `shown` / `cap` in enrichment and emit `ctx.enrich.truncated(...)` when a result page is capped (cursor returned, or trending filled the requested limit). `bsky_search_posts` reports `hitsTotal` via `ctx.enrich.total()` when the API supplies it. Search tools' guidance notes that cursor pagination is unreliable for unauthenticated public-AppView search — refine with filters instead of paging.
- **`bsky_get_trending` `notice`** — `notice` added to the tool's enrichment schema so previously-dropped empty/constrained guidance surfaces in `structuredContent`.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner (from the 0.10.6 template) wired into the `bundle` script; strips dependency-shipped agent docs `mcpb pack` cannot exclude.
- **Dockerfile `HEALTHCHECK`** — bun-native `fetch` against `/healthz` (the slim runtime image ships no curl/wget); `org.opencontainers.image.version` stamped from the `APP_VERSION` build arg.

## Changed

- **`bsky_get_post_thread` invalid-AT-URI error** — reclassified from `InvalidParams` (JSON-RPC -32602) to `ValidationError` (-32001) for the `invalid_at_uri` reason, aligning a malformed-input rejection with the framework's input-validation code.
- **`.mcpbignore`** — dev-directory patterns (`/.claude/`, `/.agents/`, `/skills/`, toolchain files) anchored to the repo root so they no longer match nested `node_modules/` paths.
- **Packaging linter** — `scripts/lint-packaging.ts` gains the 0.10.6 post-bundle content and entrypoint-identity checks; `scripts/check-framework-antipatterns.ts` updated to match.
- `@cyanheads/mcp-ts-core` `^0.9.21` → `^0.10.6`
- `@biomejs/biome` `^2.4.16` → `2.5.0` (pinned)
- `@types/node` `^25.9.2` → `^25.9.3`
- `vitest` `^4.1.8` → `4.1.9` (pinned)
