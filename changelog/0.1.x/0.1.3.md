# 0.1.3 — 2026-06-06

## Fixed

- **Empty-result notice in `structuredContent`** — `bsky_search_posts`, `bsky_search_actors`, `bsky_get_author_feed`, and `bsky_get_follows` now declare `notice` in the enrichment schema so `structuredContent.notice` surfaces on zero-result responses ([#1](https://github.com/cyanheads/bluesky-mcp-server/issues/1))
- **`bsky_get_trending` `link` field** — relative paths (e.g. `/profile/…`) are now normalized to full `https://bsky.app/…` URLs before returning ([#2](https://github.com/cyanheads/bluesky-mcp-server/issues/2))
- **Cursor 403 on unauthenticated search** — input and output `cursor` descriptions on `bsky_search_posts` and `bsky_search_actors` now disclose the public AppView limitation (cursor-based pagination returns 403 for unauthenticated requests) and point to `bsky_get_author_feed` / `bsky_get_follows` as the reliable paging tools ([#3](https://github.com/cyanheads/bluesky-mcp-server/issues/3))
