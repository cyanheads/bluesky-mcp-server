# 0.1.5 — 2026-06-20

Maintenance pass adopting `@cyanheads/mcp-ts-core` `^0.10.6 → ^0.10.9` (deps, skills, scripts). No tool-surface changes.

## Changed

- **`devcheck` packaging gate** — adopts the 0.10.9 `lint:packaging` plugin-marketplace manifest checks via the new `devcheck.config.json` `packaging.pluginManifests: true` flag: validates non-empty descriptions and scoped/unscoped identity across `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, and `.codex-plugin/mcp.json`. ([mcp-ts-core #240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`check-dependency-specifiers` devcheck step** — new `scripts/check-dependency-specifiers.ts` hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json` dependency sections and the `bun.lock` workspace map, catching a `latest` dist-tag that would let the next install re-resolve a pinned dep. ([mcp-ts-core #246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **`.codex-plugin/plugin.json`** — populated the empty `interface.longDescription` (the description the 0.10.9 manifest check now requires).
- **Fresh-scaffold devcheck robustness** — `scripts/build-changelog.ts`, `scripts/devcheck.ts`, `scripts/check-framework-antipatterns.ts`, and `scripts/check-skill-versions.ts` re-synced to the 0.10.7 versions that guard against missing changelog files, absent `.git`, and worktree-deleted skills. ([mcp-ts-core #242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243), [#237](https://github.com/cyanheads/mcp-ts-core/issues/237))
- **Project skills re-synced** — 14 `skills/<name>/SKILL.md` files mirrored from the framework, including the seven version-bumped skills (`api-auth`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `report-issue-local`, `tool-defs-analysis`) and the `ctx.content` image/audio collector docs in `api-context`. ([mcp-ts-core #238](https://github.com/cyanheads/mcp-ts-core/issues/238), [#239](https://github.com/cyanheads/mcp-ts-core/issues/239))
- `@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.9`
- `@types/node` `^25.9.3` → `^26.0.0`
